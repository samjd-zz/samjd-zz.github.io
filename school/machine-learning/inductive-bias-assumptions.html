What Models Actually Assume

Understanding Inductive Bias: The Hidden Assumptions Behind Machine Learning

üéØ The Core Concept

Inductive bias is what the model assumes about the pattern it's trying to learn before seeing the data. It's like giving the model "hints" about what kind of answer to look for.

Think of it as the model saying: "Out of infinite possible patterns that could fit this data, I'm going to assume it's probably shaped like THIS..."

ü§ñ What Each Model Assumes

Linear Regression

  • "The relationship is a straight line"
  • "Changes are proportional"
  • "Features add up independently"
  • "No weird curves or bends"

Decision Trees

  • "I can split data into rectangles"
  • "Yes/no questions find patterns"
  • "Different regions = different rules"
  • "Sharp boundaries are OK"

Neural Networks

  • "Complex = layers of simple"
  • "Features combine in intricate ways"
  • "Patterns have hierarchy"
  • "Non-linear is the norm"

k-Nearest Neighbors

  • "Similar inputs ‚Üí similar outputs"
  • "Locality matters most"
  • "Distance = similarity"
  • "Neighbors vote on the answer"

SVM

  • "Classes have a gap between them"
  • "Maximum margin is best"
  • "Boundary points matter most"
  • "Transform can linearize"

Random Forest

  • "Many simple trees > one complex"
  • "Averaging reduces errors"
  • "Different features for different trees"
  • "Voting improves accuracy"

üè† Real Example: House Prices

The Same Problem, Different Assumptions:

Linear Model assumes:
"Each bedroom adds $20,000, each bathroom adds $15,000, each 100 sq ft adds $10,000 - everywhere, always, predictably."

Tree Model assumes:
"IF location='downtown' AND size>2000sqft THEN price=$500k+, ELSE IF location='suburbs' THEN different rules apply..."

Neural Network assumes:
"Location affects how size impacts price, which interacts with age, and these combine in complex non-linear ways I'll figure out through layers."

KNN assumes:
"This house should cost about the same as the 5 most similar houses nearby."

The Key Insight

The model is essentially assuming:
"The truth is probably one of the patterns I'm designed to find"

üîß Common Biases We Encode

Smoothness

"Similar inputs should give similar outputs - no sudden jumps"

Sparsity

"Only a few features really matter - most are noise"

Locality

"What matters is what's nearby - distant things don't affect each other"

Hierarchy

"Complex patterns are built from simpler sub-patterns"

Invariance

"The pattern stays the same under certain transformations"

Periodicity

"Patterns repeat in cycles - what happened before will happen again"

Remember:

The art of machine learning is choosing biases that match reality closely enough to help, but aren't so restrictive that they prevent learning the true pattern. It's about giving the model the right "hints" without giving it the wrong "constraints".
